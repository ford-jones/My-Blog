REFLECTIONS

What is main?

The main branch is the "home" directory. All of the polished, flawless code resides there. Edits to code shouldn't happen here as you risk creating faulty code in the home directory, the main is where code that has been edited in a branch will be merged into after polishing following a pull request.

Why create a Branch?

To work on a mirror image of the main directory safely without the potential of creating bad code / syntax in the main directory.

Do the concepts introduced feel intuitive or difficult to understand?

Yeah they feel pretty intuitive, it makes sense that you wouldn't work in the main branch. You could ruin everything!!

What are some examples of when you would fork?

When you want to work on a set of code without risking any changes to the main repository, e.g. open source or public projects. You can also use it if you want a template / starting point for your own project.
